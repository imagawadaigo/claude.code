## エージェント別 最重要指摘

### クリティック（論破役）

**致命的問題 優先度順：**

1. **幹事コストが高すぎて起動しない**
   家族は職場と違い「断っても罰則がない」。招待→投稿のCVRを最初に計測できる設計にしないと、他の施策が全部無意味になる。
   → CVR 50%超が成立の条件。30%以下なら演出も価格も意味なし。

2. **演出を見る前に課金させている**
   「届ける」押下時点では演出を見ていない。価値が見えない状態での課金は離脱を招く。
   → 課金前にデモ動画/プレビューアニメを必ず見せる。

3. **花言葉クイズは3ヶ月で模倣される**
   差別化の核心は「感情の解像度を上げる体験設計」であり、クイズはその入口に過ぎない。クイズ単体を守ろうとするな。

4. **家族はクローズドで拡散しない**
   唯一の拡散経路は「幹事の"やった感"のSNS投稿」。完成ページのシェア画像自動生成が必要。

5. **5週間で全機能は無理。捨てることを決めろ。**
   クローズドβで「招待→投稿CVR」だけを計測する最小プロダクトを作ることが正解。

---

### マーケター（市場・訴求）

1. **ターゲットに「今年こそのトリガー」が必要**
   「20〜35歳・地方出身」だけでは弱い。親が高齢になった・自分が結婚した・親が病気をした、など「今年こそ」のきっかけを持つ人を狙う。

2. **人数ベース課金が招待数を減らすリスク**
   「招待しすぎると高くなる」心理が生まれ、プロダクトの核心価値（家族全員で作る）を幹事が自ら毀損する。**フラット価格（¥500〜¥600一律）を検討。**

3. **花言葉クイズの名称問題**
   「クイズ」は軽く聞こえる。「気持ちを選ぶ」「贈りたい言葉を選ぶ」に変える。

4. **受取人から口コミが生まれる設計が抜けている**
   「母が子どもたちへのお礼メッセージを送れる」機能を追加すると、それが家族LINEに流れて他の人の目に触れる口コミ導線になる。

5. **最重要問い**
   > 「なぜLINEアルバムやGoogleフォト（無料）ではなく、¥500〜¥800払ってこのサービスを使うのか」を5人以上に聞いてから開発を始めるべき。

---

### ロジシャン（ビジネスモデル検証）

1. **課金タイミングが離脱リスク最大点**
   「完成してから払わされた」は心理的抵抗が最大化するタイミング。
   修正案：招待リンク発行時に前倒し課金。先払いで「元を取ろう」心理が働き招待行動が促進される。

2. **年間120組試算が設計と矛盾**
   閑散期（7〜翌4月）に月8組の根拠がない。
   修正値：**90組・¥48,000（Stripe手数料後）**が根拠ある数字。

3. **Supabase Storageが33組で上限に達する**
   写真1枚3MB × 5枚/組 = 15MB → 500MBで約33組分。
   → **写真を1枚1MB以下にリサイズする処理を最初から実装必須。**

4. **受取りページにCTAが存在しない**
   最も感情が高まっているタイミングで「あなたも誰かに贈る」ボタンがない。LTV向上の最大機会を逃している。
   → 受取りページ末尾に「あなたも贈る」CTAを追加。`referrer_page_id`でトラッキング。

5. **LTV構造の欠落**
   年1回¥600の単発。翌年リマインドメール（created_atから1年後にトリガー）だけでリピート率が変わる。

### ユーザー代表（28歳・東京在住の長男）

**総合評価：「使いたい。¥600は安い。罪悪感を解消するプロダクトとして刺さっている。」**

1. **最も欲しかった機能（設計にない）：リマインド通知**
   「妹さんがまだ投稿していません」という通知。自分が妹を追いかけなくて済むなら、それが一番助かる。これがないとこのサービスを使う動機が半減する。

2. **花言葉クイズは「上乗せ機能」として評価**
   使う動機の本体ではなく上乗せ。ただし母が受け取ったときの「自分だけの花」演出は強い。

3. **SNS投稿はしない。でも友人にはLINEで教える。**
   「友達に教える」ボタンがあればLINEで送る。

4. **不安3点：**
   - 妹が参加しなかったら1人だけになるか
   - 母のブラウザでアニメが「真っ白」にならないか
   - 永続保存の実態（3年後もリンクは生きているか）

## 5エージェント共通の結論

### 致命的な設計上の問題（開発前に解決必須）

**① 課金タイミング：2つの意見が分かれている**

| エージェント | 意見 |
|---|---|
| クリティック | 演出を見せてから課金（後払い維持） |
| ロジシャン | 招待発行時に前倒し（先払い） |
| エグゼキューター | 一律¥500・シンプルに |

→ **決断が必要：先払い or 後払い。** 後払いはVidDayが実証済み（★4.8）。ただしロジシャンの指摘通り「完成後に払わされた感」のリスクあり。**デモ動画をLP最上部に置くことを前提に後払い維持**が現実的。

**② 招待→投稿のCVRが最重要KPI**
全エージェント共通。これが50%を下回ると全ての施策が無意味になる。

**③ リマインド通知を追加する**
ユーザー代表が「最も欲しい」と言い、マーケターも「招待後の参加率問題」を指摘。MVPの最小機能に入れるべき。
