# MVP仕様書 — 家族共同メッセージページ

> 母の日：2026年5月11日（残り約5週間）に間に合わせるための最小仕様。

---

## 鉄則：今回作らないものを決める

**作らない：**
- ユーザー登録・ログイン機能（不要。URLベースで完結）
- 通知メール（LINEで幹事が手動共有する）
- 編集・削除機能（送信後は変更不可でいい）
- 多言語対応
- 管理画面（最初はDB直接確認で可）

**作る：**
- ページ作成フォーム（幹事用）
- 投稿フォーム（参加者用、スマホ完結）
- 表示ページ（受け取る側用、動的演出）
- 決済（Stripe）

---

## 画面一覧（最小）

### 1. トップページ（ランディングページ）

- サービスの説明（3行以内）
- 「ページを作る」ボタン → 決済へ

### 2. 決済ページ

- Stripe Checkoutを使う（自前実装不要）
- ¥500 or ¥1,000プランを選ぶ
- 決済完了 → 幹事用の「編集URL」と「招待URL」が発行される

### 3. 参加者投稿ページ（招待URLを開いた先）

- 名前入力（テキスト）
- 写真アップロード（1枚）
- メッセージ入力（テキストエリア、200文字程度）
- 「送る」ボタン

### 4. 表示ページ（母・父に送るURL）

- タイトル「〇〇へ、みんなからのメッセージ」
- 投稿が順番にフェードインで現れる
- 全員分の写真＋名前＋メッセージが並ぶ
- 永続URL（ページが消えない）

## 技術スタック案

| 要素 | 選択肢 | 理由 |
|---|---|---|
| フロントエンド | Next.js（App Router） | Vercelで無料デプロイ、サーバーコスト最小 |
| DB | Supabase（Postgres） | 無料枠で十分、ファイルストレージも一体 |
| 画像保存 | Supabase Storage | S3より設定が楽 |
| 決済 | Stripe Checkout | 自前UI不要、最速で実装可能 |
| デプロイ | Vercel | 無料枠で十分 |

## データモデル（最小）

```
pages
  id (uuid)
  title (text) — 「〇〇へのメッセージ」
  event_type (text) — "mothers_day" | "fathers_day" | etc.
  organizer_name (text)
  edit_token (uuid) — 幹事用の編集URL
  view_token (uuid) — 表示用URL
  invite_token (uuid) — 参加者招待用URL
  plan (text) — "basic" | "premium"
  paid (boolean)
  created_at

messages
  id (uuid)
  page_id (uuid) — foreign key
  contributor_name (text)
  message_text (text)
  photo_url (text)
  created_at
```

## 体験フロー（詳細）

```
幹事
  1. トップページで「ページを作る」をタップ
  2. 誰宛て（例：お母さん）・節目（母の日）を入力
  3. ¥500を決済
  4. 「招待URL」をコピーしてLINEで兄弟に送る
  5. 当日、「表示URL」を母に送る

参加者（兄弟）
  1. LINEの招待URLをタップ
  2. 名前・写真・メッセージを入力して送信
  3. 終わり（所要3〜5分）

母・父
  1. 当日URLを受け取る
  2. 開くと写真とメッセージが順番に現れる
  3. 保存される → 何度でも見返せる
```
