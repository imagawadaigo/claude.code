## 価値設計の一本筋

**「なぜ使うか」**：バラバラに住む家族全員のメッセージを、感動体験として届けられる唯一の手段だから。
**「なぜ無料代替手段（LINEアルバム等）ではないか」**：動的演出・個人への印象可視化・永続保存は他にない。
**「なぜお金を払うか」**：作ったページを「届ける」体験に価値がある。デモ動画で価値を先に見せてから払う。

---

## 無料／有料の設計

| フェーズ | 状態 | できること |
|---|---|---|
| **無料** | 作成〜収集 | ページ作成・招待URL発行・投稿収集・プレビュー（演出なし） |
| **有料（一律¥500）** | 届ける | 受取り用URLの発行・フェードイン演出・永続保存 |

**価格：一律¥500（MVPはシンプルに。人数ベース3段階は削除）**

理由：人数が増えると高くなる設計は「招待しすぎると損」という心理を生み、プロダクトの核心価値（家族全員で作る）を幹事自身が毀損する。

---

## 最重要KPI：招待→投稿のCVR

**CVR 50%超が成立の条件。これを最初に計測する。**

- 招待URLを受け取った人が実際に投稿完了した割合
- 30%以下なら演出も価格も関係なく失敗する
- Week1〜2でこの数値を計測できる設計にする

## 画面一覧（最小）

### 1. トップページ（LP）
- **デモGIF/動画を最上部に配置**（これがないと課金されない）
- 「無料でページを作る」ボタン
- 3ステップ説明（作る→集める→届ける）

### 2. ページ作成フォーム（無料）
- 誰宛て（名前）
- 節目の種類（母の日 / 父の日 / 誕生日 / その他）
- 幹事の名前
- 作成 → 幹事用URLが発行

### 3. 幹事ダッシュボード
- 招待URLをコピーしてLINEに貼る
- 「誰々が記載済み」「〇〇がまだです」の一覧表示（名前ベース）
- プレビュー（演出なし・透かし入り）
- 「このページを届ける（¥500）」ボタン

### 4. 参加者投稿ページ（招待URL先）
- 名前入力（テキスト）
- 写真アップロード（1枚・1MB以下にリサイズ処理）
- メッセージ入力（200文字程度）
- 「送る」ボタン

### 5. Stripe Checkout
- 一律¥500
- 決済完了 → 受取り用URLが即時発行
- 幹事に「このURLを当日母に送ってください」と表示

### 6. 受取り体験ページ（有料後）
- 投稿が1枚ずつフェードインで現れる
- タップで全文・写真を大きく表示
- 末尾に「あなたも誰かに贈る」CTA ← 追加（LTV設計）
- 永続URL

## セキュリティ設計

| URL種別 | 用途 | トークン |
|---|---|---|
| 幹事用URL | ページ管理・支払い | UUID v4 |
| 招待URL | 参加者の投稿 | UUID v4（別トークン） |
| 受取り用URL | 母・父が開く | UUID v4（決済完了後に発行） |

- 受取り用URLは決済完了まで存在しない
- 決済ボタン押下前：「発行後はメッセージの編集ができません」の断り書きを表示

## データモデル（更新版）

```
pages
  id (uuid)
  title (text)                  — 「〇〇へのメッセージ」
  event_type (text)             — "mothers_day" | "fathers_day" | etc.
  organizer_name (text)
  edit_token (uuid)             — 幹事用URL
  invite_token (uuid)           — 招待用URL
  view_token (uuid | null)      — 受取り用URL（決済前はnull）
  paid_at (timestamp | null)
  referrer_page_id (uuid | null) — 口コミ経路トラッキング ← 追加
  created_at (timestamp)

messages
  id (uuid)
  page_id (uuid)
  contributor_name (text)
  message_text (text)
  photo_url (text)              — リサイズ済み（1MB以下）
  created_at (timestamp)
```
