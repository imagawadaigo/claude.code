# MVP仕様書 — 家族共同メッセージページ

> 母の日：2026年5月11日（残り約5週間）に間に合わせるための最小仕様。

---

## 価値設計の一本筋

**「なぜ使うか」**：バラバラに住む家族が、一つの場所に気持ちを持ち寄れる唯一の手段だから。  
**「何ができるか」**：写真＋メッセージを家族みんなで集めて、母・父に動的なページで届けられる。  
**「なぜスクショでは代替できないか」**：体験の核心がアニメーション・インタラクションにあるため、静止画では再現できない。  
**「なぜお金を払うか」**：集まった気持ちを「届ける」体験に価値がある。プレビューで確認してから払う。

## 無料／有料の設計

| フェーズ | 状態 | できること |
|---|---|---|
| **無料** | ページ作成〜投稿収集 | ページ作成・招待URL発行・兄弟の投稿収集・プレビュー（演出なし） |
| **有料** | 送信・公開 | 受け取り用URLの発行・オープニング演出・モーダル体験・永続保存 |

## 課金モデル：参加人数ベース

スクショで済ませられない設計にするほど、人数が多いほど体験価値が高い。

| 参加人数 | 価格 | 想定ユースケース |
|---|---|---|
| 2〜3人 | ¥300 | 兄弟2人 + 幹事 |
| 4〜6人 | ¥500 | 4〜6人きょうだい |
| 7人以上 | ¥800 | 大家族・親族 |

- 支払いは幹事1人（参加者は無料）
- 人数は投稿が集まった段階でカウントされる（事前登録不要）

## 受け取り体験UX設計（スクショ対策の核心）

### オープニング：花束アニメーション
- ページを開くと花束が画面中央にパッと広がる演出
- 静止画・スクショでは再現できないモーションが体験の入口

### メッセージ一覧：カードレイアウト
- 花束が引いた後、投稿者ごとのカードが順番にフェードイン
- 各カード：投稿者の写真サムネイル + 名前 + メッセージ冒頭（省略）

### 個別メッセージ：モーダル展開
- カードをタップ → モーダルが開く
- モーダル内：写真（大）+ 名前 + メッセージ全文
- 「閉じる」で一覧に戻る
- 何度でもタップして見返せる

### なぜスクショでは不十分か
- 花束アニメーションは動画でしか伝わらない
- カードのタップ体験・モーダルの質感はインタラクションそのもの
- 全員分のカードを一画面に収めると小さすぎて読めない
- **「開くたびに体験できる」**こと自体が価値（LINE上のスクショは一度きり）

## 共有フロー：LINEで個人に送る

```
幹事 → 受け取り用URLをコピー → LINEで母（または父）個人に送る
```

- グループチャットではなく**個人LINEで送る**ことを推奨（サプライズ性を保つ）
- URLをLINEに貼ると、OGPでタイトル・サムネイルがプレビュー表示される
- OGP設計：「〇〇さんへ、家族みんなからのメッセージが届いています」

---

## セキュリティ設計

URLベースで完結させるが、推測不可能なトークンで保護する。

| URL種別 | 用途 | トークン仕様 |
|---|---|---|
| 幹事用URL | ページ管理・支払い | UUID v4（128bit、推測不可） |
| 招待URL | 参加者の投稿 | UUID v4（別トークン） |
| 受け取り用URL | 母・父が開く | UUID v4（決済完了後に発行） |

- 受け取り用URLは**決済完了まで存在しない**（払っていない人は届けられない）
- 各URLは独立したトークンで、幹事URLから他URLへの推測はできない

## 鉄則：今回作らないもの

**作らない：**
- ユーザー登録・ログイン（URLトークンで代替）
- 通知メール（LINEで幹事が手動共有）
- 編集・削除機能（送信後は変更不可）
- 多言語対応
- 管理画面（最初はDB直接確認で可）
- メッセージ文章のテンプレ・コピー（それ自体で完結してしまい、払わなくなる）

**作る：**
- ページ作成フォーム（幹事用）
- 参加者投稿フォーム（スマホ完結）
- 幹事ダッシュボード（投稿確認・プレビュー・送信ボタン）
- 受け取り体験ページ（花束演出 → カード → モーダル）
- 決済（Stripe Checkout、人数ベース）

## 画面一覧（最小）

### 1. トップページ
- サービスの概要（3行以内）
- 受け取り体験のデモGIF or 動画（花束〜モーダルまで）
- 「無料でページを作る」ボタン

### 2. ページ作成フォーム（無料）
- 誰宛て（名前）
- 節目の種類（母の日 / 父の日 / 誕生日 / その他）
- 幹事の名前
- 作成 → 幹事用URLが発行

### 3. 幹事ダッシュボード
- 招待URLをコピーしてLINEに貼る
- 誰が投稿したか一覧表示（人数カウント）
- プレビュー（演出なし・透かし入り）
- 「このページを届ける（¥〇〇）」ボタン（人数に応じた金額を表示）

### 4. 参加者投稿ページ
- 名前入力
- 写真アップロード（1枚）
- メッセージ入力（200文字程度）
- 「送る」ボタン

### 5. Stripe Checkout
- 人数に応じた金額（¥300 / ¥500 / ¥800）
- 決済完了 → 受け取り用URLが発行

### 6. 受け取り体験ページ（有料完了後）
- 花束アニメーション
- カード一覧（フェードイン）
- タップ → モーダル展開

## 技術スタック

| 要素 | 選択肢 | 理由 |
|---|---|---|
| フロントエンド | Next.js（App Router） | Vercelで無料デプロイ |
| DB | Supabase（Postgres） | 無料枠で十分・ストレージ一体 |
| 画像保存 | Supabase Storage | S3より設定が楽 |
| 決済 | Stripe Checkout | 人数ベース課金に対応可能 |
| アニメーション | Framer Motion | React向け・花束・フェードイン実装が楽 |
| デプロイ | Vercel | 無料枠で十分 |

## データモデル

```
pages
  id (uuid)
  title (text)                  — 「〇〇へのメッセージ」
  event_type (text)             — "mothers_day" | "fathers_day" | etc.
  organizer_name (text)
  edit_token (uuid)             — 幹事用URL
  invite_token (uuid)           — 招待用URL
  view_token (uuid | null)      — 受け取り用URL（決済前はnull）
  plan_price (integer | null)   — 300 / 500 / 800（決済前はnull）
  paid_at (timestamp | null)
  created_at (timestamp)

messages
  id (uuid)
  page_id (uuid)
  contributor_name (text)
  message_text (text)
  photo_url (text)
  created_at (timestamp)
```
