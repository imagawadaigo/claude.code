# MVP仕様書 — 家族共同メッセージページ

> 母の日：2026年5月11日（残り約5週間）に間に合わせるための最小仕様。

---

## 価値設計の一本筋

**「なぜ使うか」**：バラバラに住む家族が、一つの場所に気持ちを持ち寄れる唯一の手段だから。  
**「何ができるか」**：写真＋メッセージを家族みんなで集めて、母・父に届ける動的なページを作れる。  
**「なぜお金を払うか」**：作ったページを実際に「届ける」体験に価値があるから。プレビューで良さを確認してから払う。

---

## 無料／有料の設計

| フェーズ | 状態 | できること |
|---|---|---|
| **無料** | ページ作成〜投稿収集 | ページ作成・招待URL発行・兄弟の投稿収集・プレビュー確認 |
| **有料（¥500〜1,000）** | 送信・公開 | 受け取り用URLの発行・アニメーション演出・永続保存 |

**なぜこの設計か：**
- 無料段階で「みんなのメッセージが集まった状態」を幹事が目で確認できる
- 「これを届けたい」という感情が最高潮になった瞬間に決済が走る
- 払った人だけが受け取り体験（動的演出・永続URL）を得られる
- 払っていない人は「プレビュー状態」のまま止まる（内容は見えるが届けられない）

---

## セキュリティ設計

URLベースで完結させるが、推測不可能なトークンで保護する。

| URL種別 | 用途 | トークン仕様 |
|---|---|---|
| 幹事用URL | ページ管理・支払い | UUID v4（128bit、推測不可） |
| 招待URL | 参加者の投稿 | UUID v4（別トークン） |
| 受け取り用URL | 母・父が開く | UUID v4（決済完了後に発行） |

- 受け取り用URLは**決済完了まで存在しない**（払っていない人は届けられない）
- 各URLは独立したトークンで、幹事URLから他URLへの推測はできない

## 鉄則：今回作らないもの

**作らない：**
- ユーザー登録・ログイン（URLトークンで代替）
- 通知メール（LINEで幹事が手動共有）
- 編集・削除機能（送信後は変更不可）
- 多言語対応
- 管理画面（最初はDB直接確認で可）

**作る：**
- ページ作成フォーム（幹事用）
- 参加者投稿フォーム（スマホ完結）
- 幹事ダッシュボード（誰が投稿したか確認・プレビュー・送信ボタン）
- 受け取り体験ページ（動的演出）
- 決済（Stripe Checkout）

## 画面一覧（最小）

### 1. トップページ（ランディングページ）

- 「何ができるか」が3秒でわかる
- 体験イメージの動画 or GIF（実際の受け取り画面のデモ）
- 「無料でページを作る」ボタン

### 2. ページ作成フォーム（幹事用・無料）

- 誰宛て（名前）
- 節目の種類（母の日 / 父の日 / 誕生日 / その他）
- 幹事の名前
- 作成 → 幹事用URLが発行される（メモ or ブックマーク必須の案内）

### 3. 幹事ダッシュボード（幹事用URL）

- 招待URLをコピーしてLINEに貼る
- 誰が投稿したか一覧表示
- プレビューボタン（受け取り体験の確認、透かし表示）
- 「このページを届ける（¥500）」ボタン → Stripe Checkoutへ

### 4. 参加者投稿ページ（招待URLを開いた先）

- 名前入力
- 写真アップロード（1枚）
- メッセージ入力（200文字程度）
- 「送る」ボタン

### 5. 決済（Stripe Checkout）

- ¥500（ベーシック）/ ¥1,000（プレミアム：豪華アニメーション）
- 決済完了 → 受け取り用URLが発行される
- 幹事に「このURLを当日送ってください」と表示

### 6. 受け取り体験ページ（有料完了後に発行）

- タイトル「〇〇へ、みんなからのメッセージ」
- 投稿が1枚ずつフェードインで現れる
- 全員分の写真＋名前＋メッセージが並ぶ
- 永続URL（消えない）

## 技術スタック

| 要素 | 選択肢 | 理由 |
|---|---|---|
| フロントエンド | Next.js（App Router） | Vercelで無料デプロイ、サーバーコスト最小 |
| DB | Supabase（Postgres） | 無料枠で十分、ファイルストレージも一体 |
| 画像保存 | Supabase Storage | S3より設定が楽 |
| 決済 | Stripe Checkout | 自前UI不要、最速で実装可能 |
| デプロイ | Vercel | 無料枠で十分 |

## データモデル

```
pages
  id (uuid)
  title (text)                  — 「〇〇へのメッセージ」
  event_type (text)             — "mothers_day" | "fathers_day" | etc.
  organizer_name (text)
  edit_token (uuid)             — 幹事用URL
  invite_token (uuid)           — 招待用URL
  view_token (uuid | null)      — 受け取り用URL（決済前はnull）
  plan (text | null)            — "basic" | "premium"（決済前はnull）
  paid_at (timestamp | null)    — 決済完了日時
  created_at (timestamp)

messages
  id (uuid)
  page_id (uuid)                — foreign key
  contributor_name (text)
  message_text (text)
  photo_url (text)
  created_at (timestamp)
```

## 体験フロー（詳細）

```
幹事
  1. トップページで「無料でページを作る」をタップ
  2. 誰宛て・節目を入力 → 幹事用URLが発行
  3. 「招待URL」をLINEで兄弟に送る
  4. ダッシュボードで誰が投稿したか確認
  5. プレビューで全員分確認 →「届ける（¥500）」をタップ
  6. Stripe決済 → 受け取り用URLが発行
  7. 当日、受け取り用URLを母に送る

参加者（兄弟）
  1. LINEの招待URLをタップ
  2. 名前・写真・メッセージを入力して送信（3〜5分）

母・父
  1. 当日URLを受け取る
  2. 開くと写真とメッセージが順番に現れる
  3. 何度でも見返せる
```
