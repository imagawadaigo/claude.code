## 1. 花タイプ診断（`flower_type`）× 開花アニメーション（MVPコア）

### コンセプト：他己診断
MBTIは自己診断だが、これは**他己診断**。
幹事が「送る相手はどういう人か」を答えることで、相手の花タイプが決まる。

受取る側は「家族が自分をこう見ていたんだ」という発見と、その花が咲く演出で感動する。
**「あなたを花に例えると」ではなく、「家族があなたに選んだ花」という構造。**

### 内部コード名
`flower_type`（機能名・DB カラム名・コンポーネント名に使う）

### UI文言の方向性
- **幹事側（作成時）**：「お母さんって、どんな人ですか？」という問いかけ
  - MBTI的な設問ではなく、印象・記憶・感情を引き出す3問
- **幹事への結果表示**：「お母さんの花が決まりました」（花名 + 花言葉）
  - 「受け取る本人には秘密です」と添える
- **受取り側（オープニング）**：「みんながあなたに選んだ花が咲きます」
  - 花が咲く → 「花言葉：〇〇」が表示
  - 「家族が見たあなたは、〇〇のような人でした」

### 機能名候補（まだ決定していない）
→ 「あなたの花」はキャッチーさが足りない。別途決定する。

### クイズ設計（案）

| 質問 | 選択肢 |
|---|---|
| あなたのお母さんを一言で表すと？ | 太陽のような存在 / 静かな強さがある / いつも笑顔 / 芯が強い |
| お母さんと過ごした思い出の場所は？ | 台所・家の中 / 公園・自然の中 / どこか遠く / 近所の商店街 |
| お母さんに伝えたい気持ちは？ | ありがとう / ごめんね / 大好き / 尊敬している |

### 花の対応表（案）

| 花 | 花言葉 | 演出の雰囲気 |
|---|---|---|
| カーネーション | 「母への愛・感謝」 | 温かいピンク・やわらかい光 |
| バラ | 「愛・尊敬」 | 深みのある赤・ゆっくり開花 |
| ひまわり | 「あなただけを見つめる・憧れ」 | 明るい黄色・光が広がる |
| アジサイ | 「家族の絆・一致団結」 | 複数の小花が集まって広がる |
| チューリップ | 「思いやり・ありがとう」 | パステル・柔らかいフェードイン |
| ユリ | 「純粋・威厳」 | 白・静かで神聖な演出 |
| 桜 | 「精神の美しさ・優しさ」 | 花びらが舞い散る |
| コスモス | 「乙女の純潔・謙虚」 | 揺れる・風を感じる演出 |

### 体験の流れ

```
幹事
  クイズに答える（3問・30秒）
  → 「あなたのお母さんには〇〇の花が選ばれました」（でも受け取る側には秘密）
  → ページに花の種類は記録されるが、母には「どんな花か」は表示されない

受け取る母
  URLを開く
  → 「さあ、どんな花が咲くのでしょう」
  → オープニングアニメーションで花がパッと咲く
  → 「この花言葉は〇〇です」と表示
```

### 差別化としての強度
- Memoreelに絶対ない
- 「相手のことを考えてクイズに答えた」という行為そのものが愛情表現
- 受け取る側に「私のために選んでくれた花」という特別感が生まれる
- 演出のバリエーションが花の種類分だけ自動的に生まれる

## 2. 日時指定解禁

### 機能
- ページ作成時に「解禁日時」を設定できる
- 解禁前にURLを開くと：カウントダウン表示（「〇日〇時間後に届きます」）
- 解禁と同時に：自動的にオープニングアニメーションが起動

### 活用パターン

**パターンA：数日前に予告送信**
```
5月8日：「母の日に届くプレゼントを送ります」
→ URLを送る → 開くとカウントダウン
5月11日 19:00：自動解禁 → 花が咲く
```

**パターンB：時刻指定サプライズ（リアル連動）**
```
幹事が事前にリアルのプレゼントを用意しておく
「19時になったら、棚の引き出しを見て」とLINEで送る
→ 引き出しにプレゼント + 「このURLも開いてみて」メモ
→ URLを開くと花が咲いてメッセージが届く
```
**デジタルとリアルを組み合わせた最高のサプライズ設計。**

### 技術的実装
- `unlock_at (timestamp)` をDBに追加
- 受け取りページで `unlock_at` と現在時刻を比較
- 解禁前：カウントダウンコンポーネントを表示
- 解禁後：アニメーション開始（ページをリロードするか、setIntervalで自動検知）
