## 1. 原体験：なぜこの事業をやるのか

### 出発点
中学3年生の時、友人に言われた一言。
「今川君って、なんで自分が正しいと思っているの？」

この言葉で気づいた。自分は誰かと「ちゃんと向き合って」きたことがなかった。
周りの人の背景や想いを想像すること、人の痛みに敏感になれる自分になりたいと思い始めた。

### 被災地での体験
祖父の「現場に行って、自分で考えられる人間になれ」という言葉に背中を押され、
中学3年生で東北被災地3県をヒッチハイクで一人でまわった。

**石巻・チーム王冠（チームオーカン）での体験：**
- 津波で妻を亡くしたUさんが「誰かに聞いてもらえることがありがたい」と語った
- おちゃっこバス：孤立した高齢者をバスに乗せてお茶会を開く活動
- 普段話していない高齢者が、場が与えられた瞬間に生き生きと話し出す
- その瞬間に立ち会えたことが嬉しかった

**陸前高田で見た「復興の矛盾」：**
- 整備された街なのに人がいない。街ができるまでに時間がかかりすぎて、住民は別の土地に根を張っていた
- 復興計画は外部コンサルと行政が作り、地元の声が反映されなかった
- 寿司屋の大将：「気仙大工の伝統を活かした温かい街にできたはずなんだ」

**ここから得た信念：**
- 支援とは「してあげること」ではなく「寄り添うこと」
- 復興とは「モノ」ではなく「心を再び繋ぎ直す営み」
- 人は「場」さえあれば、自分の言葉で話し始める

### 事業との接続
家族への気持ちはある。でも「伝える場」がないから伝えられていない。
フラワーポートレートとメッセージページは、その「場」を作ること。
届いた瞬間に何かが動く——その場に立ち会えるプロダクトを作っている。

## 2. 良し悪しの判断基準

### 温度で判断する

| 判断 | 状態 | 例 |
|---|---|---|
| **良い** | 血が通っている。照れくさい本音が引き出され、関係性が前より温かくなる | フラワーポートレートで「相手のことを考えた」行為が生まれる。受取った人が「こう見てくれていたんだ」と感じる |
| **悪い** | 無機質。やっただけで終わる。温度感がない | テンプレ文を貼って完了。感動もリアクションもない。LINEで送るのと変わらない |

### 3つの判断テスト

すべての機能・設計判断に対して：

1. **きっかけテスト**：この機能は「照れくさい感情を引き出すきっかけ」になるか？
2. **温度テスト**：これを使った後、相手との関係性は温かくなるか？
3. **双方向テスト**：「やった」で終わるか、「伝わった」で終わるか？双方向のリアクションが生まれるか？

### やらないプロダクト・やらない機能

- **代筆・自動生成**：本人の言葉じゃないものに温度はない（Cocoro Letterとの明確な違い）
- **テンプレートで完結するもの**：テンプレを選んで送るだけで「やった感」が出てしまう
- **効率化が目的のもの**：「早く済ませる」ための機能は本質的コミュニケーションの逆
- **匿名の仕組み**：誰が書いたか分からないメッセージに温度は生まれない

## 3. ターゲットへの解像度：誰の、どんな感情に触れたいか

### 表面の状態
「照れくさくて言えないまま時間が過ぎている人」

### 根にある感情構造
照れくささの**手前**に、もっと深い層がある。

```
否定された経験・劣等感・自分はダメだという自己認識
  → 弱さを抱えている状態
    → 気持ちを差し出す勇気が出ない
      → 言いたいことが言えない
        → 時間だけが過ぎていく
```

- 「照れくさいから言えない」ではなく「自分に自信がないから、気持ちを差し出す勇気が出ない」
- 精神的・経済的に余裕がある人は、人に優しくできるし、気持ちを届けられる
- 弱さを抱えている人は、その余裕がないから言えない

### このプロダクトが介入するポイント

弱さそのものを治すプロダクトではない。
**弱さを抱えたままでも「伝えられる場」を作る**プロダクト。

- フラワーポートレートの3問は「自分の言葉で感情を書け」とは言わない
- 「お母さんってどんな人？」という問いかけで、自然に相手のことを考える行為が始まる
- 選択肢を選ぶだけで、すでに「本音のコミュニケーション」が発生している
- ハードルを下げることで、弱さを抱えた人でも参加できる設計

### 理想の体験後の状態

プロダクトを使った後に起きてほしいこと：
- 受取った人が「こう見てくれていたんだ」と気づく（他己診断の力）
- 送った側が「伝えてよかった」と感じる
- そこから会話が始まる（「あの花、なんで選んだの？」）
- 関係性が、プロダクトを使う前より一段温かくなっている

## 4. 問題構造（CLAUDE.mdより精緻化）

### 理想の社会
お互いにギブしすぎて「もらいすぎだ」と感じるほど温かい心が循環している社会。
人が自分の「光」（強み・可能性・隠れた感情）に気づき、精神的・経済的に自立した上で、
他者に圧倒的にギブできる状態が当たり前になっている。

### 現実の問題
```
体験の蓄積（否定・失敗・比較）
  → 自分が劣っていると感じる（劣等感の可視化）
    → 弱さを隠す / 本音を出せなくなる
      → 人に優しくする余裕がなくなる
        → 関係性が冷たくなる / 疎遠になる
          → さらに孤立する
```

### この事業が解くレイヤー
全体を解くのではなく、**「弱さを抱えたままでも、大切な人に気持ちを届けられる」**という一点に集中する。

- 劣等感そのものは解かない
- 自己肯定感を上げるプロダクトではない
- 「場」を作ることで、弱くても参加できるようにする
- 参加して「伝わった」という成功体験が、結果的に自信につながる可能性はある（が、それは副次効果）

## 5. ミッションとの一貫性チェック

> ミッション：「人として一番大事な人間関係を、より温かいものにする」

すべての設計判断で、以下を問う：

- この機能は**人間関係の温度を上げるか**？
- この設計は**弱さを抱えた人が参加できるか**？
- この体験の後に**双方向のリアクションが生まれるか**？
- 被災地のおちゃっこバスと同じ構造——**「場」を用意すれば、人は自分の言葉で話し始める**——をデジタルで実現しているか？

これらに「はい」と答えられないものは、ミッションから外れている。
