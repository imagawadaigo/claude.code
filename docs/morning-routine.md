# 朝ルーティン仕様書

最終更新：2026年4月

---

## 概要

毎朝「おはよう」と声をかけるだけで、ニュース摂取・思考整理・その日のタスク決めまでを Claude と一緒にこなすルーティン。

---

## トリガー

ユーザーが **「おはよう」** と発話したとき、以下のステップを順番に実行する。

---

## ステップ構成

### STEP 1：ニュース配信

WebSearch を使い、**当日の**ニュースを3本取得して提示する。

| カテゴリ | 検索テーマ例 |
|---|---|
| ① ビジネス・経済 | 国内外の注目企業動向、マーケット、スタートアップ資金調達 |
| ② 日本社会・政策 | 若者・働き方・テクノロジー政策、少子化・地方創生など |
| ③ テクノロジー・AI | AI新機能、注目プロダクト、起業家の動き |

**フォーマット：**
```
📰 STEP 1：今日のニュース

① [カテゴリ名]
見出し：○○○○
要約：（2〜3行）
→ なぜ知っておくべきか：（1行コメント）

② ...
③ ...
```

エンタメ・プロ野球は含めない。

---

### STEP 2：ニュースについてのチャット & プロファイリング

- ニュースに対してユーザーが思うことを話してもらう
- 「どう思う？」「自分の事業と接続するとしたら？」など深掘りを促す
- 一区切りついたら → 「Notionに保存しますか？」と確認
- 保存する場合 → Notion MCP経由で **「朝ルーティンログ」DB** に追記

---

### STEP 3：今日のスケジュール確認

Google Calendar MCP 経由で当日のイベントを取得・表示する。

```
📅 STEP 3：今日のスケジュール

[時刻] イベント名
[時刻] イベント名
...

空き時間：XX:XX〜XX:XX（約N時間）
```

---

### STEP 4：タスク決め & カレンダー登録

1. Notion の **「タスク在庫」DB** からステータスが「未着手」のタスクを取得
2. 優先度・工数を照らし合わせ、「今日やること」を一緒に決定
3. 決定したタスクを Google Calendar に時間ブロックとして登録

```
🎯 STEP 4：今日やること

今日の空き時間を踏まえて、以下を提案します：

□ タスク名（工数目安：N時間）
□ タスク名（工数目安：N時間）

これでよければカレンダーに入れます。変更あれば教えてください。
```

---

## Notion データベース構造

### DB①：朝ルーティンログ

| フィールド名 | 種別 | 内容 |
|---|---|---|
| 日付 | Date | 当日の日付 |
| ニュース① | Text | 見出し + 要約 |
| ニュース② | Text | 見出し + 要約 |
| ニュース③ | Text | 見出し + 要約 |
| 気づき・感想 | Text | チャットで出てきた発言・思考 |
| キーワード | Multi-select | 気づきのタグ（例：AI・起業・政策） |

### DB②：タスク在庫

| フィールド名 | 種別 | 内容 |
|---|---|---|
| タスク名 | Title | やること |
| カテゴリ | Select | 事業 / 大学 / その他 |
| 優先度 | Select | 高 / 中 / 低 |
| 工数目安 | Number | 時間（例：1.5） |
| ステータス | Select | 未着手 / 進行中 / 完了 |
| 期限 | Date | 任意 |
| メモ | Text | 背景・補足 |

---

## MCPの前提

| ツール | 用途 | 環境 |
|---|---|---|
| Notion MCP | 朝ルーティンログ保存、タスク在庫参照 | ローカルClaude Codeのみ |
| Google Calendar MCP | 当日予定取得、タスクのカレンダー登録 | ローカルClaude Codeのみ |
| WebSearch | ニュース取得 | どのセッションでも可 |

> **注意：** Claude Code web セッション（サーバー側）では Notion・Google Calendar MCP が使えない。
> ローカルの Claude Code CLI または デスクトップアプリから起動したときのみ、STEP 2〜4 の外部連携が動作する。
